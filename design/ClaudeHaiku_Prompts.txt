## Plan: Static Scope Variable Resolution

### TL;DR
Implement a **paragraph-scoped variable symbol table** that validates variables during single-pass parsing. Exit immediately on first "use-before-definition" error without generating output, matching Rust/Java error behavior.

---

### **Steps**

**Phase 1: Symbol Table Infrastructure**
1. Create `VariableSymbolTable` struct with global & local scope tracking
2. Add `current_line` tracking to lexer (increment on `\n`)
3. Add symbol table & line tracking fields to `Parser`

**Phase 2: Scope Management**
4. Insert `enter_scope()` at start of `parse_paragraph()`
5. Insert `exit_scope()` at end of `parse_paragraph()` (after #MKAY)
6. Track current line number during parsing

**Phase 3: Variable Definition** *(parallelizable with Phase 4)*
7. Modify `parse_variable_define()` to:
   - Extract VARNAME and VARVALUE tokens
   - Call `symbol_table.define(varname, line, value)` 
   - Allow redefinition (overwrites previous)

**Phase 4: Variable Usage Validation** *(parallelizable with Phase 3)*
8. Modify `parse_variable_use()` to:
   - Extract VARNAME token
   - Call `symbol_table.lookup(varname)`
   - Return `Err("Error: Variable 'X' used before definition")` if not found
   - Continue on success

**Phase 5: Error Propagation**
9. Update `main()` to handle parse errors:
   - Propagate `parse_lolcode()` result
   - Exit with code 1 and error message on failure
   - Skip output generation if errors found

**Phase 6: Output Guard**
10. Skeleton for output generation (actual generation is future work)

---

### **Relevant Files**

- main.rs — Single implementation file:
  - Lines 20-210: `CharLexicalAnalyzer` — Add `current_line` field
  - Lines 252-290: `Parser` struct — Add `symbol_table` field
  - main.rs: `parse_variable_define()` — Extract & store VARNAME/VARVALUE
  - main.rs: `parse_variable_use()` — Lookup & validate VARNAME
  - main.rs: `parse_paragraph()` — Add scope enter/exit
  - main.rs: `main()` — Handle parse errors & guard output

---

### **Verification**

- ✓ Variable defined in body, used in body → Success
- ✓ Variable defined in paragraph P1, used in P1 → Success  
- ✓ Variable defined in P1, used in P2 → Error (out of scope)
- ✓ Variable used before definition → Error: "Variable 'X' used before definition"
- ✓ Variable redefined → Uses new value (no error)
- ✓ Semantic errors found → Exit code 1, no output file

---

### **Design Decisions**

| Decision | Choice | Rationale |
|----------|--------|-----------|
| **Scoping** | Paragraph-scoped (body-level = global) | Useful isolation; simpler than document-wide |
| **Redefinition** | Allowed (overwrites) | Simpler implementation |
| **Analysis** | Single-pass during parsing | No need for separate AST/second pass |
| **Error Format** | Minimal: `"Variable 'X' used before definition"` | Clear and sufficient per requirements |
| **Exit Policy** | First error → exit(1), no output | Match Rust/Java compiler behavior |

---

### **Further Considerations**

1. **Future: Rich Error Messages** — Track definition line number; report `"defined at line X"` when validating usage
2. **Future: Variable Substitution** — Extend to replace variables with values in output generation (separate phase)
3. **Future: Type System** — Validate variable definitions and usages against declared types
